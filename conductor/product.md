# Product Guide

## 產品願景
建立一個現代化且功能齊全的中央導覽中心（Central Hub），將使用者在 GitHub 上的所有專案及對應的 GitHub Pages 連結整合在一個直觀的介面中。這不僅是個人的作品展示窗，更是開發者同儕探索其技術成果的入口。

## 目標受眾
- **開發者與技術同儕：** 觀察專案結構、程式碼品質及技術堆疊。
- **一般大眾：** 瀏覽感興趣的成品與實驗性專案。

## 核心功能
- **自動化專案列表：** 整合 GitHub API 或自動化邏輯，識別並列出所有開啟 GitHub Pages 的儲存庫。
- **搜尋與篩選：** 提供即時搜尋與分類篩選功能，讓訪客能快速定位特定類型的專案（如 Web 應用、工具類等）。
- **半自動化管理：** 結合自動抓取與手動設定檔（JSON/YAML），允許使用者自定義顯示順序、隱藏特定專案或添加補充描述。
- **現代化儀表板 UI：** 使用卡片式佈局展示專案，並包含預覽圖與簡短說明，提供優質的視覺體驗。

## 使用者體驗 (UX) 目標
- **直觀導覽：** 訪客應能在幾秒鐘內理解網站目的並找到所需的專案連結。
- **視覺吸引力：** 採用現代化設計規範，確保在不同裝置上都有良好的呈現效果。
